# evidence/

Multi-device screenshot evidence captured by testeador's `DeviceFleet`.

## Layout

Every multi-device step writes one subdirectory keyed by `label`:

```text
evidence/<label>/
  android-emulator-5554.png      # one PNG per device, same wall-clock
  android-emulator-5556.png
  ios-F7B5XXXX.png
  composite.png                  # all of the above side-by-side, with headers
  metadata.json                  # label, timestamp UTC, dims, skewMs, paths
```

## What to look at

`composite.png` is the canonical artifact. It is the single image an AI agent
or human reviewer opens to decide whether a multi-device step looked right.
The per-device PNGs are the raw source; `composite.png` is the summary.

`metadata.json` is the machine-readable record of the same information,
including the `skewMs` field — the wall-clock spread between the fastest and
slowest device shot. A small skew (< 100 ms on M-series Macs) means the
composite is a fair side-by-side comparison; a large skew suggests one device
was slower to redraw, and the composite may mix states.

## Why it is not committed

PNGs and especially composites are large and regenerated on every run. The
`.gitignore` keeps only this README and itself. Tests and the CLI rewrite the
folder on demand.

## Generated by

- `dart test test/contract_test.dart -N streaming` (smoke flow with composites
  per step).
- `dart run bin/snapshot_fleet.dart <label> [device-id ...]` (ad-hoc capture).
- Any testeador `TestStep.action` calling `fleet.snapshotComposite(label)`.
